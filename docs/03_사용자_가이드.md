## 1. 시작하기

### 1.1 필수 요구사항

- Node.js 18.x 이상
- npm 9.x 이상
- 모던 웹 브라우저 (Chrome, Firefox, Safari, Edge)

### 1.2 프로젝트 클론

```bash
git clone https://github.com/[username]/zlsu.git
cd zlsu
```

---

## 2. 로컬 개발 환경 설정

### 2.1 의존성 설치

```bash
npm install
```

### 2.2 개발 서버 실행

```bash
npm run dev
```

실행 후 브라우저에서 접속:
```
http://localhost:5173/zlsu/
```

### 2.3 프로덕션 빌드

```bash
npm run build
```

빌드 결과물은 `dist/` 폴더에 생성됩니다.

### 2.4 빌드 미리보기

```bash
npm run preview
```

## 3. 테스트 방법

### 3.1 테스트 계정

| 구분 | 이메일 | 비밀번호 |
|------|--------|----------|
| 관리자 | admin@swimteam.com | admin123 |
| 일반회원 | user1@test.com | user123 |

### 3.2 기능별 테스트 시나리오

#### 시나리오 1: 비로그인 사용자 테스트

1. **메인 페이지 확인** (`/`)
   - 정원 현황 표시 확인 (현재 5/16명)
   - "가입 신청", "로그인" 버튼 표시 확인

2. **회칙 조회** (`/rules`)
   - v2.0 (현재 회칙) 내용 확인
   - v1.0 버튼 클릭하여 이전 회칙 확인

3. **가입 신청** (`/apply`)
   - 모든 필드 입력 후 신청
   - 이메일 중복 체크 확인
   - 비밀번호 6자 이상 검증 확인

#### 시나리오 2: 일반 회원 테스트

1. **로그인** (`/login`)
   - user1@test.com / user123 입력
   - 로그인 성공 후 메인 페이지 이동 확인

2. **내 정보 확인** (`/my`)
   - 이름, 이메일, 상태 등 정보 확인
   - 닉네임, 연락처 수정 테스트

3. **회원 목록 조회** (`/members`)
   - 전체 회원 목록 표시 확인
   - 상태별 필터 기능 테스트

4. **상태 변경 신청** (`/change-status`)
   - 상태 선택 및 사유 입력
   - 신청 완료 확인

5. **로그아웃**
   - 헤더의 로그아웃 버튼 클릭
   - 메인 페이지 이동 및 비로그인 상태 확인

#### 시나리오 3: 관리자 테스트

1. **로그인** (`/login`)
   - admin@swimteam.com / admin123 입력

2. **관리자 홈** (`/admin`)
   - 대기 중인 신청 건수 확인
   - 정원 현황 확인

3. **가입 신청 관리** (`/admin/applications`)
   - 대기 중인 신청 목록 확인
   - 승인 테스트: "승인" 클릭 → 회원으로 추가 확인
   - 반려 테스트: "반려" 클릭 → 사유 입력 → 상태 변경 확인

4. **회원 관리** (`/admin/members`)
   - 전체 회원 목록 확인
   - 상태 변경 신청 승인/반려 테스트
   - 회원 수정 페이지 이동

5. **회원 수정** (`/admin/members/:id`)
   - 회원 정보 수정 테스트
   - 상태 직접 변경 테스트
   - 탈퇴 처리 테스트

### 3.3 데이터 초기화

로컬 스토리지를 초기화하면 테스트 데이터가 리셋됩니다:

```javascript
// 브라우저 개발자 도구 콘솔에서 실행
localStorage.clear();
location.reload();
```

## 4. 일반 사용자 가이드

### 4.1 가입 신청하기

1. 메인 페이지에서 "가입 신청" 버튼 클릭
2. 다음 정보 입력:
   - 이름 (필수)
   - 이메일 (필수, 로그인 ID로 사용)
   - 비밀번호 (필수, 6자 이상)
   - 연락처 (필수)
   - 생년월일 (선택)
   - 가입 동기 (필수)
3. "가입 신청" 버튼 클릭
4. 관리자 승인 후 로그인 가능

### 4.2 로그인하기

1. 메인 페이지에서 "로그인" 버튼 클릭
2. 이메일과 비밀번호 입력
3. "로그인" 버튼 클릭

### 4.3 내 정보 수정하기

1. 로그인 후 헤더에서 "내정보" 클릭
2. "수정" 버튼 클릭
3. 닉네임, 연락처 수정
4. "저장" 버튼 클릭

### 4.4 활동 상태 변경 신청하기

1. 내 정보 페이지에서 "상태 변경 신청" 클릭
2. 변경할 상태 선택 (활동/비활동/휴식)
3. 변경 사유 입력
4. "신청하기" 버튼 클릭
5. 관리자 승인 후 상태 변경

## 5. 관리자 가이드

### 5.1 관리자 대시보드

- 경로: `/admin`
- 대기 중인 가입 신청 건수 확인
- 대기 중인 상태 변경 신청 건수 확인
- 현재 정원 현황 확인

### 5.2 가입 신청 관리

- 경로: `/admin/applications`
- 대기/승인/반려 필터로 신청 조회
- **승인**: 회원으로 자동 등록
- **반려**: 반려 사유 입력 필수

### 5.3 회원 관리

- 경로: `/admin/members`
- 상태별 회원 필터링
- 상태 변경 신청 승인/반려
- 개별 회원 정보 수정

### 5.4 회원 정보 수정

- 경로: `/admin/members/:id`
- 수정 가능 항목: 이름, 닉네임, 연락처, 생년월일, 상태, 역할
- 수정 불가 항목: 이메일 (로그인 ID)
- **탈퇴 처리**: 상태를 '탈퇴'로 변경
- **삭제**: 회원 데이터 완전 삭제 (복구 불가)

## 6. 문제 해결

### 6.1 로그인이 안 되는 경우

1. 이메일/비밀번호 확인
2. 가입 신청이 승인되었는지 확인
3. 탈퇴 처리된 계정인지 확인

### 6.2 페이지가 안 보이는 경우

1. URL 경로 확인 (base path: `/zlsu/`)
2. 브라우저 캐시 삭제 후 새로고침
3. 개발 서버가 실행 중인지 확인

### 6.3 데이터가 저장 안 되는 경우

1. 브라우저의 로컬 스토리지가 활성화되어 있는지 확인
2. 시크릿/프라이빗 모드인지 확인
3. 로컬 스토리지 용량 초과 여부 확인

### 6.4 개발 서버 에러

```bash
# node_modules 삭제 후 재설치
rm -rf node_modules
npm install

# 캐시 삭제
npm cache clean --force
```

### 6.5 빌드 에러

```bash
# TypeScript 에러 확인
npm run build

# 에러 메시지에 따라 해당 파일 수정
```

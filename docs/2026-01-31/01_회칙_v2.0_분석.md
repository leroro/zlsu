## 2. 회원 레벨 체계 비교

### 현재 시스템 (`constants.ts`)
```
1. 뉴비(newbie): 가입 후 4회 미만 참여
2. 일반(regular): 최근 2개월간 월 2회 이상 참여
3. 열정(passionate): 최근 2개월간 월 3~4회 참여
4. 핵심(core): 최근 3개월간 월 3~4회 이상 꾸준히 참여 + 대회 참석 의향
5. 스태프(staff): 모임 운영 업무/기여
```

### 새 회칙 v2.0
```
1. 뉴비: 4회 미만 참여
2. 일반: 월 1~2회 참석
3. 열정: 월 3~4회 참석 (거의 풀참)
4. 핵심: 열정회원 + 각종 이벤트 참여 (번개 모임, 대회 등)
5. 스텝: 운영진
```

### 차이점
- **일반**: "월 2회 이상" → "월 1~2회" (기준 완화)
- **핵심**: "대회 참석 의향" → "각종 이벤트 참여" (범위 확대)
- 용어: "스태프" → "스텝" (표기 통일 필요)

## 3. 휴면 제도 (신규 규정)

### 새 회칙 v2.0 휴면 규정

**휴면 = 단톡방에는 남아있지만 활동은 쉬는 상태 (회비 없음)**

| 구분 | 내용 |
|------|------|
| 신청 단위 | 월 단위 (예: 2~3월 휴면) |
| 신청 방법 | 총무(미선)에게 전달 말일까지 직접 연락 |
| 자진 사전 신청 | 전달 말일까지 신청 → **2달까지 자리 보전** |
| 미납 시 강제 휴면 | 매달 7일까지 회비 미납 → 강제 휴면, **자리 즉시 공석** |
| 복귀 조건 | 활동 회원 정원(14명)에 자리가 있어야 함 + 해당 월 회비 납부 |

### 시스템 영향
- 현재 시스템에는 "자리 보전" 개념이 없음
- 휴면 회원의 복귀 우선순위 관리 기능 필요할 수 있음
- 단, 현재는 수동 운영으로 충분할 수 있음 (향후 검토)

## 5. 향후 검토 사항 (시스템 기능 추가)

### 5.1 휴면 자리 보전 관리
- 자진 휴면 vs 강제 휴면 구분
- 자리 보전 기간 관리 (2달)
- 복귀 대기열 관리

### 5.2 회비 미납 자동 처리
- 매달 7일 기준 미납 회원 감지
- 강제 휴면 처리 알림

### 5.3 참석/불참 알림 연동
- 현재 시스템 외 카카오톡 일정으로 운영 중
- 시스템 연동은 향후 검토

## 6. 게스트 관련

회칙 v2.0에는 "게스트" 개념이 있으나, **회원 관리 시스템에서는 게스트를 관리하지 않음**.

- 게스트는 단톡방에서 수동으로 관리
- 시스템에는 정식 회원(pending/active/inactive/withdrawn)만 존재
- 당일 참여 우선순위(활동→휴면→게스트)는 운영자가 판단

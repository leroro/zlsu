1. 모임 정원 기준
- 팀 인원이 늘어날 경우 코치님께 부담될 수 있고, 트윈 눈치도 보여 적정 정원에 대한 기준 필요
=> 모임 정원 관리기능 필요

2. 회원 자격 유지 기준
- ‘쉰다’는 상태에 대한 기준 논의 필요 (예: 3주 이상 미참여 시 비활동 기간 미리 신청 등)
- 회비를 납부하며 쉬는 경우와, 회비를 납부하지 않고 쉬는 경우가 혼재되어 있어 회원 자격을 어떻게 유지·구분할지에 대한 기준 필요 (예: 비활동 회원 기준)
=> 가입신청중/활동/휴면/휴면해제신청중/탈퇴
=> 관리자 승인 기능 필요

3. 신규 회원 수용 기준
- 어떤 경우에 초대할지 기준이 명확하지 않음 (예: 정원 미달 시 + 추천인 + 수영실력 등 일정 조건 충족 시)
=> 정원 미달시 어떻게 처리될지에 대한 로직 있나? 가입신청서 입력폼에 '약관동의', '개인정보수집동의'가 없음(회칙), 추천인, 수영실력 확인 폼이 없음 예) 자유형/배영/평영/접영

4. 입회하지 않은 참여에 대한 기준
- 입회하지 않고 참여하는 경우 허용 여부 기준 정리가 필요함 (예: 활동회원 n인 미달시 1회 체험 허용, 초과시 입회 필요 등)
=> 이건 약관에 명시하면 됨. 초안이 필요

5. 회비 월 납입 관리
- 월 회비 납입이 일관되게 관리되지 않고 있어 미납 시 적용 기준과 정기 안내 알림 등 관리 방식 정리가 필요함 (예: 자동 운영 기능이 있는 팀 카톡방 신규 개설, 이동 후 매월 1일 회비 캘린더 알림 자동 발송, 총무가 1주 후 미납자 재확인 알림 수동 발송 등)
=> 이것도 약관에 명시. 알림 수락 동의 내용에 필요

6. 운영 부담 경감
- 기준이 모호하면 관리 부담이 커질 수 있으므로 최대한 단순·명확하게 가져가기
- 자동화 가능한 부분은 자동화할 예정이며, 자동화 어려운 운영 역할은 분담에 대한 논의도 필요함
=> 회원목록에 스탭 표시와 어떤 역할인지 표시하는 기능 있었으면.

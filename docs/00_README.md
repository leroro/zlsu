# 문서 디렉토리 안내

## ⚠️ 중요: 단일 진실 공급원 (SSOT)

**이 폴더의 문서는 참고용/아카이브입니다.**

**시스템의 최신 정보는 아래 두 파일이 기준입니다:**

1. **InfoPage** (가입 전 비회원용)
   - 파일: `src/components/about/AboutContent.tsx`
   - 내용: 모임 소개, 활동 안내, 가입 방법, 수모 안내 등

2. **회칙** (가입 후 회원용)
   - 파일: `public/rules/v2.0.md` (최신 버전)
   - 내용: 운영 규정, 회비, 시간 준수, 벌금, 정원, 회원 레벨 등

**원칙:**
- 모임 정보 변경 시 위 두 파일 먼저 수정
- docs 폴더 문서는 위 두 파일과 일치해야 함
- 불일치 발견 시 소스 코드가 우선

---

## 📂 폴더 구조

### 루트 문서 (현재 유효)
- `01_배포_가이드.md` - GitHub Pages 배포 가이드
- `02_회원가입_프로세스.md` - 회원 가입 프로세스 안내

### 루트 전략 문서 (활성 프로젝트)
진행 중인 프로젝트의 전략/설계 문서 (완료 시 아카이브 또는 삭제)
- `03_회칙_v2.0_시스템_전략.md` - 회칙 v2.0 기반 시스템 구현 전략

### backlog/ (미래 구현 예정 기능)
미구현 기능 관련 설계 문서 모음
- `issue-database-setup/` - Supabase DB 설정
- `issue-sns-login/` - SNS 로그인

### YYYY-MM-DD/ (날짜별 작업 아카이브)
각 세션별 작업 내역 및 개발 문서
- `2026-01-21/` - 초기 MVP 개발 문서
- `2026-01-22/` - 이중 승인 시스템 개발
- `2026-01-23/` ~ `2026-02-08/` - 이후 세션별 작업 내역

## 📝 문서 작성 규칙

1. **새 문서 작성 시**
   - InfoPage(`AboutContent.tsx`) 또는 회칙(`v2.0.md`) 내용 기준으로 작성
   - 불일치 시 소스 코드 우선

2. **날짜별 폴더**
   - 세션 작업 내역은 `YYYY-MM-DD/NN_세션_작업내역.md` 형식

3. **넘버링 규칙**
   - 루트 문서: `01_`, `02_` 등 숫자 prefix
   - 날짜 폴더 내: 필요시 `01_`, `02_` 사용

---

## 🔄 세션 종료 시 문서 정리 규칙

**end-session 루틴에서 반드시 따를 것:**

1. **세션 작업내역 작성**
   - `YYYY-MM-DD/NN_세션_작업내역.md` 형식으로 작성
   - 완료 작업, 변경 파일, 다음 작업(TODO) 포함

2. **활성 프로젝트 문서 갱신**
   - 루트 전략 문서가 있으면 진행 상황 업데이트 (Phase 완료 여부 등)
   - 새 프로젝트 시작 시 루트에 넘버링 문서로 추가하고 이 README에 등록

3. **backlog 정리**
   - 활성 프로젝트에 흡수된 backlog 항목은 제거
   - 새로 발견된 미래 작업은 backlog에 추가

4. **이 README 최신 유지**
   - 루트 문서가 추가/삭제되면 이 파일의 폴더 구조 섹션 업데이트
   - 프로젝트 완료 시 "활성 프로젝트" 항목 제거

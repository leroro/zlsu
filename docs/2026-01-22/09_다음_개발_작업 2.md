# 다음 개발 작업 목록

## 1. UI 공통
- 화면이동시 맨 위로 스크롤이 올라 가야 되는데 지금 전혀 그렇게 되어 있지 않음. 스크롤이 밑에 중간 부터 시작함. 가입만이 아니라 라우팅 기본 설정이 그렇게 되어야 하나테스트 브라우저는구글 크롬 IOS 에서 확인했을 때문제가 있었어요

## 2. 가입 관련
- 최초 가입 승인 이후 안내 단계 필요
   - 대시보드 할 일 섹션에 팀 카톡방에 입장하라는 플로우 추가해야하지 않을지?
   - 카톡방에 입장하고 자기소개한 다음에, 일정 확인하라는 안내문이 나와줘야함함

- 가만 생각해보니 회원가입 첫 번째 단계는 '추천인 확인'부터 시작 해야 되는 거 아닌가? 이게 뒤에 나오면 골치아파짐. 단계 재설계 필요.

- 최초 회원은 추천인이 없는데 대체 어떻게 가입 하라는 거지? 그 부분에 대한 로직을 검토 하지 못했다

- 가입 안내 화면에
   - 절차 콘텐츠 잘못 안내됨 : 총무가 초대하는게 아니라 로그인 후 카톡방 입장하기 버튼 클릭해야 함.
   - CTA 버튼 플로팅 처리 : 내용이 길어서 가입 신청하기 버튼이 하단에 상시 고정되게 처리하면 더 좋겠음. 맨 하단에 내려가면 원래 UI대로 유지되고.

- 가입 완료 화면 : 내용이 예전걸로 되어있음. 추천인 동의 후 관리자가 입금 확인한 뒤 최종 가입 승인 하는건데, 지금 그런 안내가 아니고 예전 안내문이 잔존함.

## 3. 대시보드
- 소개 페이지 링크 복사해서 공유할 수 있는 기능이 있으면 좋겠음. 잔여석 안내하는 부분에 초대 링크 복사하기 버튼 추가하면 좋을듯?

- 자주 찾는 메뉴에 카톡방 입장 버튼이 안보임. 그거는 권한 관계없이 고정적으로 항상 보여야 되는데 잘못 구현됨.

## 4. 생일
- 음력 회원 생일은 생일 명단에서 양력 기준으로 표시되도록 개선
   - 예를들어 정상옥님 생일을 음력 2/22로 세팅하고, 해당 회원의 생일이 4/9(목)으로 나오고 있는지 확인 필요
   - 음력 생일인 사람은 따로 표시도 되어야 할 것 같음

2. 사람들이랑 찍은 사진도 추가하면 좋을듯 함.

# 에이전트 베이스 템플릿

## 용도

새 프로젝트에서 에이전트를 빠르게 설정하기 위한 **범용 템플릿**입니다.
프로젝트 전용 맥락(용어, 도메인, 기술 스택)은 포함되어 있지 않습니다.

## 사용법

### 새 프로젝트에 적용할 때

```bash
# 1. 새 프로젝트의 .claude 폴더 생성
mkdir -p 새프로젝트/.claude/agents
mkdir -p 새프로젝트/.claude/agent-memory

# 2. 필요한 에이전트 템플릿 복사 (전체 또는 선택적)
cp .claude/agents-base/*.md 새프로젝트/.claude/agents/

# 3. 각 파일에서 "# 프로젝트 맥락 (커스터마이징 필요)" 섹션만 수정
```

### 커스터마이징 포인트

각 에이전트 파일은 두 파트로 나뉩니다:

- **범용 원칙**: 수정하지 않는다 (모든 프로젝트 공통)
- **프로젝트 맥락**: 이 섹션만 새 프로젝트에 맞게 수정

## 포함된 템플릿

### 설계 단계
| 파일 | 역할 | 추천 모델 |
|------|------|----------|
| `product-owner.md` | 방향성, 정책 판단, 우선순위 결정 | Opus |
| `spec-analyst.md` | 요구사항 분석, 시스템 설계 | Opus |
| `ux-writer.md` | UX 라이팅, 콘텐츠 구조, 인지 부하 관리 | Opus |

### 구현 단계
| 파일 | 역할 | 추천 모델 |
|------|------|----------|
| `ui-developer.md` | 프론트엔드 컴포넌트, CSS, 반응형 | Opus |
| `data-developer.md` | 타입 정의, API, 데이터 레이어 | Opus |

### 검증 단계
| 파일 | 역할 | 추천 모델 |
|------|------|----------|
| `code-reviewer.md` | 코드 품질, 보안, 시스템 일관성, UX 정합성 | Sonnet |
| `qa-tester.md` | 빌드 검증, 테스트 시나리오, 엣지 케이스 | Sonnet |

### 운영 단계
| 파일 | 역할 | 추천 모델 |
|------|------|----------|
| `devops.md` | 빌드, 배포, Git 관리 | Haiku |
| `cost-advisor.md` | 비용 분석, 가성비 최적화, 기술 부채 | Sonnet |

## 팀원에게 공유할 때

이 폴더를 포함한 `.claude/` 전체를 git에 커밋하면 됩니다.
팀원이 checkout만 하면 동일한 에이전트 설정으로 작업할 수 있습니다.

## 프로젝트 간 메모리 이전

`agent-memory/` 폴더는 프로젝트별 학습 내용이므로 새 프로젝트에 복사하지 않습니다.
에이전트가 새 프로젝트에서 작업하면서 자연스럽게 메모리를 쌓아갑니다.

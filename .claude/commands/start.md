# Start - 작업 시작

새 세션 시작 시 이전 세션의 다음 작업을 자동으로 찾아 안내합니다.

## 작업 순서

### 1단계: 최신 작업 내역 찾기
1. `docs/` 폴더에서 가장 최신 날짜 폴더 확인
2. 해당 폴더에서 가장 마지막 `*_세션_작업내역.md` 파일 찾기
3. 파일 내용 읽기

### 2단계: 다음 작업 파악
1. 파일에서 "다음 작업" 또는 "TODO" 섹션 찾기
2. 해당 내용을 사용자에게 요약하여 안내

### 3단계: 현재 상태 확인
1. `git status`로 현재 브랜치 및 변경사항 확인
2. `npm run build`로 빌드 상태 확인

### 4단계: 안내
다음 형식으로 사용자에게 안내:

```
## 이전 세션 작업 내역
- 파일: docs/YYYY-MM-DD/NN_세션_작업내역.md
- 마지막 작업: [완료된 작업 요약]

## 다음 작업 (TODO)
[다음 작업 내용]

## 현재 상태
- 브랜치: [현재 브랜치]
- 빌드: [성공/실패]

바로 다음 작업을 시작할까요?
```

## 파일 탐색 방법

```bash
# 최신 날짜 폴더 찾기
ls -d docs/2026-* | sort -r | head -1

# 해당 폴더에서 최신 세션 작업내역 찾기
ls docs/YYYY-MM-DD/*세션_작업내역.md | sort -r | head -1
```

## 주의사항

- 작업 내역 파일이 없으면 CLAUDE.md의 "다음 작업" 섹션 확인
- 빌드 실패 시 먼저 오류 해결 제안
- 사용자에게 바로 작업 시작 여부 확인

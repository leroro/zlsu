# Session Summary - 세션 작업 내역 문서화

현재 세션에서 작업한 내역을 문서로 정리합니다.

## 작업 순서

1. `git log`로 최근 커밋 내역 확인
2. `git diff --name-only HEAD~N`으로 변경된 파일 목록 확인
3. `git status`로 현재 상태 확인
4. 변경 내용을 분석하여 작업 내역 정리
5. `docs/YYYY-MM-DD/NN_세션_작업내역.md` 파일로 문서화
   - 날짜 폴더가 없으면 생성
   - NN은 해당 날짜의 다음 순번

## 문서 포맷

```markdown
# 세션 작업 내역 (YYYY-MM-DD #NN)

## 개요
작업 목적 간략 설명

---

## 완료된 작업

### 1. 작업 제목
- 상세 내용
- 변경 사항

### 2. 작업 제목
- 상세 내용

---

## 변경된 파일 목록

### 신규 파일
| 파일 | 설명 |
|------|------|
| `path/file.tsx` | 설명 |

### 수정된 파일
| 파일 | 주요 변경 |
|------|----------|
| `path/file.tsx` | 변경 내용 |

---

## 다음 작업 (TODO)

### 우선순위 높음
1. 작업 내용

### 우선순위 중간
2. 작업 내용

---

*문서 작성일: YYYY-MM-DD*
```
